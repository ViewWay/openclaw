## 🎯 改进方案

### 一、中文化（预计 13-18 周）

#### 1. CLI 命令中英文支持（3-4 周）

```bash
# 英文命令
openclaw gateway start --port 18789

# 中文命令（新增）
openclaw 网关 启动 --端口 18789
```

**关键文件**: `src/cli/program/command-registry.ts`, `src/cli/argv.ts`

#### 2. 代码注释双语化（9-12 周）

```typescript
// 改进前
// Validate the authentication token

// 改进后
// 验证认证令牌 / Validate the authentication token
```

**优先级**:

- 高: 核心模块（45 文件，2-3 周）
- 中: 业务逻辑（90 文件，4-5 周）
- 低: 辅助模块（150 文件，3-4 周）

#### 3. 错误信息/日志双语化（6-8 周）

```typescript
// 错误信息系统
ERROR_MESSAGES = {
  AUTH_FAILED: {
    code: "E002",
    en: "Authentication failed",
    "zh-CN": "认证失败",
  },
};
```

**关键文件**: `src/infra/errors.ts`, `src/logging/*.ts`

---

### 二、安全增强（预计 8-10 周）

#### 1. API 认证增强（4-5 周）

- ✅ Token 过期机制（Device Token 30 天过期）
- ✅ 多因素认证（TOTP 2FA）
- ✅ API Key 权限细粒度控制

**新增文件**: `auth-token-expiry.ts`, `auth-2fa.ts`, `auth-permissions.ts`

#### 2. 权限控制完善（5-6 周）

- ✅ RBAC 角色系统（Admin/Operator/User/Guest）
- ✅ 资源 ACL 访问控制列表
- ✅ 权限审计日志

**新增文件**: `rbac.ts`, `resource-acl.ts`, `permission-audit.ts`

#### 3. 其他安全增强（2-3 周）

- ✅ 敏感信息过滤增强（支持中文模式）
- ✅ 安全头增强（CSP、HSTS 等）
- ✅ IP 白名单增强
